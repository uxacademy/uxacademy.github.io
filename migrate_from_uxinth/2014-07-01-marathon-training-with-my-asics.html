ช่วงนี้กำลังฝึกวิ่งเพื่อไปวิ่ง เชียงใหม่มาราธอน ครับ ซึ่งการวิ่งระยะไกลแบบจริงจังครั้งล่าสุดก็คือเมื่อตอน ป. 4 จึงต้องหาตารางการซ้อมวิ่งที่เหมาะกับคนอยาก วิ่งมาราธอน มือใหม่ครับ ในหนัง รัก 7 ปี ดี 7 หน มีการอ้างถึงตารางซ้อมมาราธอนของ Jeff Galloway แต่เห็นตารางแล้วคนที่วิ่ง 2 กิโลเมตรก็หอบจะแย่อย่างเรา ก็คงขอผ่านไปก่อน

เมื่อช่วงกลางปีที่แล้วมีคนจาก AQ Works ที่เป็นคนออกแบบ My ASICS มาเยี่ยมที่ออฟฟิศ และจุดเด่นของ My ASICS คือการ customise โปรแกรมฝึกวิ่งมาราธอนให้เข้ากับปัจจัยพื้นฐานของคนวิ่งและระยะเวลาที่เราต้องการวิ่งได้ดี ด้วยโมเดลของซ้อมวิ่งที่ออกแบบโดยนักวิทยาศาสตร์ของ ASICS เอง เลยถือโอกาสลองเสียเลย ซึ่งผมคิดว่าเป็นหนึ่งใน application ที่มี UX ตอบสนองความต้องการของคนต้องการซ้อมวิ่งเป็นอย่างมาก ก็เลยสละหยาดเหงื่อแรงกายทดสอบแอปเสียเลยครับ

ก่อนอื่นก็อย่าลืมพื้นฐานว่า การออกแบบ UX ใดๆ ให้นึกถึงว่า UX เป็น "อะไรสักอย่าง" ที่จะพาผู้ใช้จาก Input ไปยัง Output ที่ต้องการให้ได้

[caption id="attachment_1205" align="aligncenter" width="300"] UX ในฐานะพาหนะที่พา User จาก Input ไปยัง Output[/caption]

เราลองมาวิเคราะห์ My ASICS ตามแผนภาพข้างบนดู

Input

[caption id="attachment_1211" align="aligncenter" width="320"] หน้าจอแรกของ My ASICS บน iPhone[/caption]

คำถามแรกที่โปรแกรมจะถามคือ คุณต้องการวิ่งระยะไหน? ซึ่งก็จะมี

  • Marathon (42.195 ก.ม.)
  • Half Marathon (21.0975 ก.ม.)
  • 10 กิโลเมตร
  • 5 กิโลเมตร
  • 10 ไมล์
  • 5 ไมล์

[caption id="attachment_1214" align="aligncenter" width="320"] หน้าจอเลือกวันวิ่ง[/caption]

หลังจากเลือกระยะวิ่งแล้วโปรแกรมจะถามว่าตั้งใจจะวิ่งวันไหน? แต่ก็มีตัวเลือกหากไม่ได้จะซ้อมเพื่อไปลงแข่ง

[caption id="attachment_1216" align="aligncenter" width="320"] หน้าจอเลือกเป้าหมายเวลาที่อยากวิ่งได้[/caption]

พอเลือกวันที่ต้องการวิ่ง ก็มาเลือกต่อว่าเวลาเป้าหมายที่ต้องการวิ่งได้คือเท่าไร อย่างกรณีนี้ ผมอยาก วิ่งมาราธอน ได้ภายใน 4:30 ชั่วโมง

[caption id="attachment_1218" align="aligncenter" width="320"] หน้าจอเลือกระยะทางที่เคยวิ่งได้[/caption]

โมเดลการคำนวนการซ้อมคือการเปรียบเทียบกับระยะทางที่เคยวิ่งได้มาก่อนดังนี้

  • Marathon (42.195 ก.ม.)
  • Half Marathon (21.0975 ก.ม.)
  • 10 กิโลเมตร
  • 5 กิโลเมตร
  • 30 - 60 นาที
  • 20 - 30 นาที
  • น้อยกว่า 20 นาที

[caption id="attachment_1221" align="aligncenter" width="320"] หน้าจอถามระยะเวลาที่เคยวิ่งได้[/caption]

พอเราเลือกระยะทางแล้วก็ต้องระบุว่าเวลาที่ระยะทางที่เราเคยวิ่งได้นั้น เราใช้เวลาวิ่งไปเท่าไร

[caption id="attachment_1222" align="aligncenter" width="320"] หน้าจอสรุปผลการคำนวณ[/caption]

พอเรามี Input หมดแล้วโปรแกรมก็จะสรุปให้นิดหน่อยก่อนที่จะบอกถึงโปรแกรมการซ้อม

Output

[caption id="attachment_1225" align="aligncenter" width="320"] หน้าจอสรุปโปรแกรมการซ้อม[/caption]

เป้าหมายหลักของ My ASICS คือ การช่วยสร้างแผนการซ้อมวิ่งให้กับผู้ใช้ ดังนั้น Output จาก Input ที่ผู้ใช้ใส่ไปคือ แผนการซ้อมวิ่งที่เหมาะสมกับผู้ใช้แต่ละคน

[caption id="attachment_1231" align="aligncenter" width="320"] ตารางการซ้อมวิ่งจาก Input ที่กรอกเข้าไป[/caption]

ในแผนการซ้อมวิ่งก็จะมีรายละเอียดให้ดูและแบ่งเป็นเฟสๆ ตามหลักการซ้อมวิ่ง ซึ่งช่วยให้มือใหม่อย่างผมมีวินัยได้มากขึ้นครับ

UX ของ Input flow

[caption id="attachment_1229" align="aligncenter" width="648"] ลำดับหน้าจอ Input ของ My ASICS[/caption]

สิ่งหนึ่งที่ผมชอบมากใน My ASICS คือ flow ของหน้าจอ Input ข้อมูลที่เป็นธรรมชาติมาก

  1. ถามความคาดหวังของผู้ใช้ ผู้ใช้อยากได้อะไร
  2. ถามสิ่งที่ผู้ใช้มีอยู่หรือทำได้ในปัจจุบัน
  3. แสดงผลบนความคาดหวังและสิ่งที่มีอยู่ของผู้ใช้

ซึ่งน่าจะเป็นแนวทางการการออกแบบแอปพลิเคชั่นที่ต้องมีการรับ Input ของผู้ใช้ได้ครับ ถ้าดูดีๆ แล้วหลักการสามข้อข้างบนมันก็น่าจะเทียบได้กับการเข้าไปหาที่ปรึกษาและจะมีชุดคำถามเหล่านี้มาถามเพื่อช่วยให้ที่ปรึกษาและผู้ใช้เข้าใจตัวเองได้และเป็นธรรมชาติมากขึ้นครับ
