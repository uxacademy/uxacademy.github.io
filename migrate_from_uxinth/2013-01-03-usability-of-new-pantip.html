เว็บ pantip ออกแบบใหม่ ผมก็ตื่นเต้นไปด้วยครับ แนวคิดของ pantip ใหม่น่าสนใจมากครับ ตั้งแต่เริ่มทำลายกำแพงของความเป็นห้องแล้วยกเรื่อง tag มาแทน แต่ก็ยังสามารถรองรับคนที่เคยชินกับของเดิมอยู่โดยยังคงมีห้องให้สบายใจ แต่ผมคิดว่าป้าๆ ที่ครอบครองห้องอยู่เห็นแบบนี้คงกรี๊ดไปพักใหญ่ (ถ้าเข้าใจ)

ผมคงวิเคราะห์ว่าเปลี่ยน concept แล้วดีหรือเปล่าไม่ได้ เพราะก็ไม่ได้ใช้มานานแต่จะลองวิเคราะห์ในเชิง Usability แทนครับผมคิดว่าช่วงนี้เป็นช่วงที่ UX Designer จะได้รับคำวิจารย์ที่มีประโยชน์มากๆ เพราะมันเป็นรูปเป็นร่างชัดเจนแล้ว

อันดับแรกคือปัญหาจากแนวทางการทำลายความเป็นห้อง

[caption id="attachment_440" align="alignnone" width="764"] ปุ่มตั้งกระทู้อยู่บนหน้าแรก[/caption]

ทีม pantip ตัดสินใจเอาปุ่มตั้งกระทู้มาไว้นอกห้อง เพื่อให้คนสามารถตั้งกระทู้ได้ง่าย และลดความยืดติดกับห้องลง ซึ่งเป็นสิ่งที่ตรงตามเป้าหมาย แต่ก็มีปัญหาตามมาครับ

[caption id="attachment_439" align="alignnone" width="798"] ระยะห่างระหว่างหน้ากระทู้ถึงปุ่มสร้างกระทู้[/caption]

ผมคิดว่าการเห็นกระทู้ที่เรียงรายอยู่จะเป็นตัวกระตุ้นให้เกิดการ post มากที่สุด แต่ปุ่ม post กลับอยู่ใกลจากกระทู้มาก ประมาณหนึ่งหน้าครึ่งสำหรับจอ MacBook Pro และถ้าผมใช้จอเล็กๆ อย่าง tablet pc คงอยู่ไกลมาก สำหรับผู้ใช้ที่มีใจจะ post อยู่แล้ว หรือคนที่ใช้งานเป็นแล้วคงไม่มีปัญหา แต่กับผู้ใช้หน้าใหม่ หรือคนที่กำลังตัดสินใจว่าจะ post ดีหรือเปล่า อันนี้คงมีผล

การเอา Action (Post) ไว้ห่างๆ นอกจากปัญหาด้านความรู้สึก ยังมีปัญหาด้านการสื่อสารด้วย สำหรับการออกแบบ UI เราจะพยายามให้คนเดาได้ว่า action ต่างๆ จะทำให้เกิดอะไรขึ้น โดยการเอา action นั้นไปไว้ใกล้ๆ กับสิ่งที่จะเกิดขึ้นมาเช่น ปุ่มลบ mail ก็จะอยู่ใกล้ๆ รายการ mail เป็นต้น ดังนั้นถ้าให้เหมาะควรเอาปุ่ม Post มาไว้ใกล้ๆ กระทู้

ทางแก้ทางแรกอาจจะทำให้ Bar เปลี่ยนห้องมันลอยขึ้นมา แล้วตามผู้ใช้ไปทุกหน้า หรืออีกทางคือย้ายมาไว้ใกล้ๆ กับ รายการกระทู้ก็ได้ครับ

การที่แนวคิดของห้องหายไปทำให้สร้างปัญหาอีกหนึ่งอย่างคือ ทาง pantip ไม่สามารถสร้าง link กลับไปยังห้องได้ หลังจากที่ผู้ใช้เข้ามาอ่านกระทู้แล้ว

[caption id="attachment_438" align="aligncenter" width="775"] หน้ากระทู้[/caption]

Pantip เลือกที่จะไม่ใส่ link กลับไปยังห้องเดิม แต่เลือกใช้วิธี Open in new windows/tabs แทน ถ้าต้องการกลับไปยังหน้าห้องก็ใช้วิธีปิด tab อันนี้สำหรับผู้ใช้ที่ใช้ tab เป็นก็จะไม่มีปัญหา แต่จะมีปัญหากับผู้ใช้จำนวนมากที่ยังไม่คุ้นกับเรื่อง tab และอาจจะสร้างความสงสัยเล็กน้อยว่ามีกรณีไหนบ้างที่จะ open in new windows แล้วก็ถ้าเปิดหลาย tab อาจจะลืมไปแล้วว่าห้องเดิมที่เคยเข้ามามันอยู่ห้องไหน

ปัญหาในส่วนของการจัดการห้องยังมีอีกหลายส่วน เช่น เมื่อเข้ามาในกระทู้จะไม่รู้ว่าตัวเองอยู่ห้องไหนต้องเดาจาก tag เอา หรือ เมื่อกดตั้งกระทู้ภายในห้องแต่ต้องมาเลือกห้องกันใหม่ไม่มีค่าปริยายให้ หรือ ผู้ใช้มือใหม่ที่ยังไม่เข้าใจ tag จะทำผิดได้ง่ายมาก ควรมีบอกว่ากระทู้นี้จะไปห้องไหนบ้าง หรือ การทำให้ตัวหนังสือชื่อห้องกับชื่อกระทู้มีขนาดใกล้เคียงกันแล้วใส่ไว้จำนวนมาก ทำให้หาห้องได้ยาก กับผมไม่แน่ใจว่าคนใช้มีพฤติกรรมการอ่านชื่อกระทู้ในหน้าแรกหรือเปล่า หรือส่วนมากเข้าไปอ่านในห้อง เป็นต้น แต่เราพักไว้ก่อน ลองมาดูเรื่อง layout ของ post น่าสนุกกว่า

[caption id="attachment_437" align="alignnone" width="789"] ภาพคำถาม ต้นฉบับ[/caption]

จะสังเกตว่าเราช่องระหว่างบรรทัดไม่เท่ากัน ในภาพจะเห็นว่ากระทู้เว้นที่ว่างให้ tag เยอะมากเหมือน tag มีความสำคัญต้องการให้คนดู ทั้งๆ ที่ส่วนที่ควรเน้นคือตัวเนื้อหา

[caption id="attachment_436" align="alignnone" width="744"] การแบ่งระยะห่างระหว่างบรรทัด ของคำถาม[/caption]

[caption id="attachment_435" align="alignnone" width="794"] ทดลองแบ่งให้เท่ากัน ทำให้แยกส่วนต่างๆ ได้ชัดเจนขึ้น[/caption]

ผมทดลองแบ่งช่องบรรทัดใหม่ โดยให้ช่องระหว่างบรรทัดเท่ากันทั้งหมดยกเว้น tag ที่เอาขึ้นไปชิด title เพราะต้องการบีบให้เหลือแค่ 3 บรรทัดแล้วแล้วบรรทัดกลางจะเด่นขึ้นมา แต่ถ้าถ้าเป็น 4 บรรทัดจะไม่มีตัวเด่น

ในส่วนของ layout นอกจากเรื่องสัดส่วนแล้ว จะยังมีตำแหน่งการวาง ผู้ post และประเภทของกระทู้ที่อยู่ทางขวามือสุด ทำให้เหมือนว่าเป็นเรื่องรอง ทั้งๆ ที่ประเภทของกระทู้ และคนที่ post น่าจะเป็นส่วนที่สำคัญมากๆ ครับ

ผมมองว่า pantip ตัวใหม่นั้นดีมาก และยังสามารถปรับให้ดีขึ้นได้อีก คงต้องใช้เวลาซักพักค่อยๆ ลองผิดลองถูกกันไป ชื่นชมที่เค้ากล้าที่จะฉีกแนวจากเดิม ยอมแลกความยากกับความยืดหยุ่นซื่งคนที่จริงจังกับการใช้ pantip น่าจะชอบ แต่ก็ไม่ยากเกินไปสำหรับมือใหม่ด้วย
