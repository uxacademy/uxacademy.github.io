การออกแบบ UX ที่ดีเราต้องรู้ข้อจำกัดของสมองกันก่อน โดยเฉพาะใครที่กำลังอยากออกแบบเว็บประเภทให้ข้อมูล เว็บสอนหนังสือ หรือโปรแกรมที่มีข้อมูลมากๆ ลองมาคำนึงถึงความจำของผู้ใช้กันครับ

อันดับแรกผมขอให้ทดลองจำข้อความต่อไปนี้

ข้อความข้างบนเป็นเรื่องราวตั้งแต่ตอนตื่นจนกลับบ้้านของคนๆ หนึ่ง พวกเราหลายคนที่มีชีวิตแบบนี้จะสามารถจดจำข้อความต่างๆ ได้ง่าย แต่ใครที่ไม่ได้มีชีวิตแบบนี้พออ่านมาถึง home จะแทบจำไม่ได้เลยว่าต่อจากกินกาแฟเราทำอะไร

หลายครั้งที่เราออกแบบ UI เราคิดโดยมีความเข้าใจในเรื่องที่เราออกแบบมาเป็นอย่างดี แต่ลืมคิดไปว่าผู้ใช้ของเรานั้นพึ่งเข้ามาเป็นครั้งแรก หรือต่อให้เข้ามาหลายครั้งก็ไม่ใช่ว่าจะใส่ใจจดจำหน้าจอของเรา ผลที่ออกมาคือเว็บส่วนมากให้ข้อมูลเยอะเกินกว่าที่ ความจำระยะสั้นของผู้ใช้จะรับได้ (short-term memory) และกลายเป็นว่าผู้ใช้จำอะไรไม่ได้เลย เหมือนที่เค้าบอกว่า "ถ้าทุกอย่างสำคัญ แสดงว่าไม่มีอะไรสำคัญเลย"

ยกตัวอย่างเช่น เว็บกระทรวงการพัฒนาสังคมและความมั่นคงของมนุษย์ และ เว็บกระทรวงศึกษา ที่ให้ข้อมูลจำนวนมากในหน้าแรก

หน้าเว็บ m-society.go.th และ moe.go.th

ในตัวอย่างเป็นเว็บที่พยายามให้ข้อมูลผู้ใช้จำนวนมากตั้งแต่หน้าแรก เมื่อผู้ใช้อ่านลงมาถึงด้านล่างเขาจะลืมแล้วว่าโครงสร้างด้านบนบั้นเป็นอย่างไร การจะกลับไปอ่านหัวข้อด้านบนจึงยากมากเพราะจำไม่ได้ว่าอยู่ตรงไหน และการเข้าถึงข้อมูลที่ต้องการในครั้งที่สองจึงแทบเป็นไปไม่ได้เช่นกัน

หลายครั้งที่เว็บต้องใส่ข้อมูลในหน้าแรกจำนวนมาก เพื่อให้แน่ใจว่าผู้ใช้จะได้รับข้อมูลความเคลื่อนไหวของข่าวสาร แต่ลืมนึกไปว่า 1. ถ้าทุกข่าวมีการเคลื่อนไหวหมด กว่าผู้ใช้จะอ่านถึงข่าวที่สำคัญก็หมดแรงซะแล้ว 2. ผู้ใช้อาจจะลืมที่อยู่ของข่าวที่ตนเองสนใจไปแล้วก็ได้ เพราะยากมากที่จะจำได้ว่าต้อง scroll กี่ครั้งถึงจะลงมาถึงกล่องข่าวที่ตนเองสนใจ 3. กรณีเลวร้ายที่สุดคือ ข้อมูลบนเว็บนั้นไม่ค่อยเคลื่อนไหว เมื่อคนเข้ามาครั้งที่สองแล้วต้องใช้พลังอย่างมากที่จะหาว่าครั้งนี้มีข้อมูลอะไรต่างจากครั้งแรก แล้วพบว่าไม่มีหรือมีน้อย คนๆ นั้นจะไม่เข้ามาหาเป็นครั้งที่สาม

นักออกแบบที่คำนึงถึง UX มักเข้าใจข้อนี้ดี แต่ไม่สามารถต้านกระแสการเมืองได้ เว็บจึงต้องออกมาเป็นดังภาพตัวอย่าง แต่นั้นเป็นการโยนความรับผิดชอบครับ คนที่เป็นผู้เชียวชาญด้าน UX จะต้องเรียนรู้การเมืองและต้องสามารถบริหารผลประโยชน์บนเว็บไซต์ได้ (ซึ่งยากและต้องใช้เวลา บางทีต้องใช้เวลาหลายปี) เหมือนที่ป้า Susan เคยบอกไว้ใน video ROI ว่า


ROI ของ UX

ต้นเหตุของงานพัฒนา software มักเกิดจากปัญหา 12 ข้อ โดยมี 3 ข้อที่เกียวกับ UX คือ 1. กำหนด requirement ไม่ชัด 2. ปัญหาด้าน communication และข้อสุดท้ายคือปัญหาเรื่อง politic ที่คนออกแบบไม่ยอมเก็บข้อมูลให้ครบทุกด้าน
