ในช่วงนี้กระแสโปรแกรมมือถือที่ไทยกำลังมาแรง นักออกแบบและนักพัฒนาก็เริ่มตื่นตัวนำ UX มาใช้ในการออกแบบกันมากขึ้น ปัญหาคือส่วนใหญ่เราจะเจอแต่เครื่องมือทำ usability testing บนเว็บไซต์แต่ไม่ค่อยมีคนพูดถึงเครื่องมือสำหรับเทสบนมือถือกันเท่าไหร่ วันนี้เลยขอมานำเสนอเครืื่องมือสำหรับใช้ทำ usability testing พร้อมรีวิวสั้นๆ 3 โปรแกรม เผื่อเป็นทางเลือกสำหรับผู้ที่สนใจค่ะ

1. Delight - http://www.delight.io

ราคา: ฟรี - $300 ต่อเดือน

Delight เน้นการทำ usability testing ออนไลน์แบบเต็มสตรีม มีฟีเจอร์เด็ดๆเช่นการ record หน้าจอพร้อมแสดงการใช้งานทุกขั้นตอน เช่น การลากนิ้ว (swipe) การคลิก ซูม และอื่นๆ และสามารถใช้กล้องหน้ามือถืออัดวีดีโอเพื่อสังเกตอารมณ์ของผู้ใช้ได้ ทำให้สามารถจินตนาการความคิดของผู้ใช้ได้ง่ายขึ้นเหมือนการเทสในสถานที่จริง นอกจากนี้ยังมีการเก็บข้อมูลและสถิติเพื่อใช้ในการวิเคราะห์อย่างละเอียด ที่สำคัญติดตั้งได้ง่าย เพียงแค่ใส่โค้ดบรรทัดเดียวลงไปในโปรแกรม ก็สามารถใช้ได้ตลอดแม้เปลี่ยน user interface ก็ตาม

ข้อดี: สามารถเก็บข้อมูลได้อย่างละเอียด มีdash boardทำให้สามารถวิเคราะห์ข้อมูลได้ลึกและง่ายขึ้น ใช้ได้เกือบทุกplatform

ข้อเสีย: ราคาสูงพอสมควรและไม่ค่อยมีการพัฒนาเพิ่มเติม

 

2. Plunk - http://www.plunkapp.com

ราคา: ฟรี

Plunk จะเน้นการทดสอบอย่างง่ายๆและรวดเร็ว โดยการอัพโหลดหน้าจอมือถือที่ต้องการทดสอบ กำหนดคำสั่งให้ผู้ใช้ และนำurlที่ได้ไปแจกจ่ายให้กับผู้ใช้ หลังจากนั้นระบบจะแสดงจุดที่ผู้ใช้กดบนหน้าจอและมีรายงานสรุปสั้นๆ

เนื่องจาก Plunk สามารถทดสอบได้เพียงหน้าเดียว และผู้ใช้ไม่สามารถใช้ gesture อื่นๆเช่นลากหรือซูมได้ จึงเหมาะในการทดสอบหน้าจอมือถือที่ไม่มีความซับซ้อนมากนัก หรือใช้ทดสอบแบบร่าง หรือwireframe ในขั้นต้นค่ะ

ข้อดี: ฟรี ใช้งานง่าย ได้ผลเร็ว ทดสอบได้ทุก platform

ข้อเสีย: ทดสอบได้หน้าจอเดียว ไม่สามารถทดสอบการใช้งาน multigesture ได้

 

3. UX Recorder - http://www.uxrecorder.com

ราคา: ฟรี - $59.99

UX Recorder เน้นการทำ usability testing เฉพาะเว็บไซต์ที่แสดงผลบนมือถือ โดยสามารถ record ได้ทุกอย่างทั้ง gestureต่างๆ เสียง หรือหน้าผู้ใช้ (ต้องมีการขออนุญาตก่อน) สามารถสร้างได้หลายโปรเจคต์เพื่อใช้ในการเทสแต่ละครั้ง

ข้อดี: ราคาไม่แพงมากและฟังก์ชั่นหลักๆครบ

ข้อเสีย: ไม่สามารถเทสappได้ ใช้ได้กับ iOS 5.0 ขึ้นไปเท่านั้น ใช้ทรัพยากรเครื่องมาก ต้องเสียบที่ชาร์จระหว่างใช้งาน

 

จะเห็นได้ว่าเครื่องมือแต่ละตัวเหมาะกับการใช้งานที่แตกต่างกัน เราจึงต้องแน่ใจก่อนว่าสิ่งที่เราต้องการจะเทสคืออะไร และเราต้องการข้อมูลแบบไหน และถึงแม้ว่าเครื่องมือทุกตัวจะสามารถใช้เทสแบบออนไลน์ และแชร์ข้อมูลได้ แต่ถ้าหากเราสามารถเทสด้วยตัวเองในสถานที่จริงได้จะดีที่สุดค่ะ เพราะเราสามารถสังเกตอารมณ์ของผู้ใช้ได้โดยตรง และสามารถถามความรู้สึกหรือสิ่งที่เราอยากรู้ได้ทันที

ในครั้งนี้เราได้เครื่องมือการทำ usability testing ไปแล้ว ครั้งหน้าจะมาต่อเร่ื่องเครื่องมือสำหรับใช้ทำ prototype บนมือถือกันค่ะ
