ได้ไปอ่านบทความของ radesign.in มาครับ เค้าเขียนเรื่อง Design lessons from Gmail web app ได้น่าสนใจมากครับ ในบทความจะบอกเราว่ามีอะไรบ้างที่เค้าสะดุดใจในความเปลี่ยนแปลงของ Gmail ตัวที่เป็น Mobile Web และที่สำคัญคือเค้าลองเสนอแนวทางมาด้วยว่าสามารถเปลี่ยนให้มันดูดีขึ้น (ในสายตาของเค้า) ได้อย่างไรบ้าง เพราะอะไรบ้าง

พออ่านแล้วพบว่าได้เรียนรู้มากมาย เลยขอจดบันทึกเอาไว้ที่นี่ด้วยเลยครับ

บทความเล่าว่า ช่วงเดือนที่ผ่านมา (Mar 2013) Google ได้ปรับหน้า Mobile web Gmail ที่ใช้เปิดบนมือถือ ให้มีหน้าตาไปในแนวทางเดียวกับ GMail  App ซึ่งทางผู้เขียน (@rakeshdesign) รู้สึกว่ามันอ่านยาก แม้ว่า Google จะทำให้มันเรียบมากขึ้นเพื่อให้เป็นไปตามแนวทางของเค้า แต่ก็มีปัญหาตามมาหลายข้อ เราลองมาดูหน้าตาตอนนี้กันก่อน

[caption id="attachment_762" align="aligncenter" width="380"] Gmail Original Screen[/caption]

ไม่แน่ใจว่าเพื่อนๆ สะดุดใจอะไรกันบ้าง แต่คุณ @rakeshdesign แยกประเด็นปัญหาออกมาเป็น 3 ข้อครับ

1. ปัญหาเรื่องการแยกแยะเนื้อหา

เค้าอ้างอิงคำพูดของคุณ Jason Cornwell (lead designer for Gmail) บอกเอาไว้ว่า

We spent a lot of time with typography to make it feel light and clean, and not like a burden

แต่ดูเหมือนบทความนี้จะไม่เห็นด้วย เค้าบอกว่า Font (Helvetica Neue) ที่ใช้ตอนนี้พอประกอบกับสีของจางๆ ของตัวหนังสือ มันทำให้ความบางของ Font สร้างความลำบากในการอ่าน ซึ่งไม่ดีเลย แต่ก็พอเข้าใจได้เพราะคุณ Cornwell เคยบอกว่า

If we just design [Gmail] to look like an Apple app, we’ve sort of failed. We want it to have some personality that’s its own.

เค้าบอกว่าไม่ต้องการให้ Gmail หน้าตาเหมือน Apple app เค้าต้องการสร้างเอกลักษณ์ของตัวเอง แต่ถ้ามันทำให้ UX แย่ลงก็ไม่ดีนะ บทความนี้ไม่ยอมแพ้เค้าบอกว่ามันน่าจะมีหนทางที่จะยังคงความเป็นตัวของตัวเองและสามารถอ่านได้ง่ายอยู่นะ

เริ่มจากผู้เขียนบทความทดลองปรับ font ในเรื่องขนาด ความหนา และสีดู

[caption id="attachment_760" align="aligncenter" width="641"] ทดลองปรับ font[/caption]

จากการทดลองข้างต้น เค้าได้นำมาปรับหน้าเว็บโดยลองปรับวิธีแยกระหว่างจดหมายที่อ่านแล้วและยังไม่ได้อ่านด้วยสีแทนที่จะเป็นตัวหนา ลองปรับ Subject ให้เข้มขึ้น และลดสีของวันเวลาลง

[caption id="attachment_769" align="aligncenter" width="608"] เทียบต้นฉบับกับแบบที่ปรับแล้ว[/caption]

ผมมองว่าแบบใหม่อ่านง่ายกว่าต้นฉบับมาก แต่มันทำให้นึกถึง Apple Mail อยู่เหมือนกัน ลองมาดูรายละเอียดที่เค้าปรับดูครับ

[caption id="attachment_774" align="aligncenter" width="632"] รายละเอียดการปรับแก้[/caption]

ผมคิดว่าถ้าเปลียนจากสีฟ้าเป็นสีแดงน่าจะช่วยให้นึกถึง Apple Mail น้อยลง แต่ด้วยความที่สีฟ้ามันมีความหมายว่า Link ในโลกของ Web มันเลยตัดสินใจลำบากอยู่ครับ ทีนี้นอกจากหน้ารายการจดหมายแล้ว หน้าเนื้อหาจดหมายก็สามารถปรับได้เช่นเดียวกันครับ

[caption id="attachment_764" align="aligncenter" width="380"] หน้าเนื้อหา[/caption]

หน้าเนื้อหาจดหมายของ Gmail อ่านง่ายกว่าหน้ารายการจดหมายเพราะทั้งขนาดและสีของ font ที่ชัดกว่า แต่ก็ยังสามารถทำให้ดีขึ้นได้อีกครับ

[caption id="attachment_767" align="aligncenter" width="380"] ขอบข้างเนื้อหา[/caption]

เบื้องต้นเราจะเห็นว่าหน้านี้ Gmail ใส่ขอบให้กับจดหมาย ซึ่งทำให้เรารู้สึกว่ามันเป็นเหมือนกระดาษแผ่นหนึ่ง แลกมาด้วยการเสียความ minimal ไปแต่ก็อย่างที่ Cornwell บอกไว้ว่าต้องการสร้างความเป็นตัวของตัวเองให้ Gmail เลยอาจจะจำเป็นต้องใส่ก็ได้ :-(

ผู้เขียนบทความมองว่า Margin ระหว่างขอบถึงตัวหนังสือมีค่ามาก การทั้งต้องเสียพื้นที่ไป 20 pixel มันไม่คุ้มกับค่า Branding เลยเค้าเลยคิดว่าต้องตัดออก แต่ก็มีคำถามว่าแล้วขอบแค่ไหนถึงจะดี เค้าเลยเล่าให้ฟังว่าในงานพิมพ์เค้าคิดกันอย่างไร

[caption id="attachment_761" align="aligncenter" width="678"] ทดลองวางระยะห่างของ column[/caption]

ก็จะเห็นว่าในรูปแรกระยะห่างระหว่าง column มีความกว้างน้อยกว่าระยะห่างระหว่างบรรทัดทำให้มันดูเป็นบรรทัดเดียวกันมากกว่าแยกเป็นสอง column แต่ถ้าเราดูรูปที่สองถ้าระยะห่างระหว่าง column มากกว่าระยะห่างระหว่างบรรทัดก็จะทำให้มันดูเป็นบทความเดียวๆ มากกว่า ในแบบที่สามเราลองทำให้เท่ากัน อันนี้ดูน่าจะใช่ที่สุด เค้าเลยเอามาปรับใช้กับ Gmail ครับ

[caption id="attachment_766" align="aligncenter" width="684"] เทียบต้นฉบับกับแบบที่ปรับแล้ว[/caption]

ผมรู้สึกว่าอ่านง่ายขึ้น หลักๆ น่าจะเป็นเพราะ font ที่ใหญ่ขึ้นส่วนเส่นที่หายไปก็รู้สึกโล่งดี มาลองดูว่าเค้าปรับอะไรไปบ้าง

[caption id="attachment_768" align="aligncenter" width="760"] แสดงรายละเอียดที่ปรับปรุงไป[/caption]

แม้ว่าระยะห่างระหว่างขอบด้านข้างจะไม่เท่ากับระยะห่างระหว่างบรรทัดแต่ก็ใกล้เคียงกันครับ ไม่รู้ว่าเพื่อนๆ คิดเห็นกันอย่างไรลอง comment ดูนะครับ สำหรับข้อที่ 2-3 ผมขอตัดไปเป็นครั้งหน้าเพราะดูเหมือนตอนนี้จะเริ่มยาวเกินไปซะแล้ว

ก่อนจะอ่านข้อที่ 2-3 อยากให้ลองอ่านบทความ เรื่องการปรับมาเป็น 2.0 ด้วยจะทำให้เราเข้าใจจิตใจคนออกแบบได้มากขึ้นครับ

[caption id="attachment_776" align="aligncenter" width="642"] หน้าจอตัวอย่างการปรับจาก 1.0 มาเป็น 2.0[/caption]
