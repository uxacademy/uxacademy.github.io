ปกติหลัง UX Workshop เราจะให้ทุกคนได้ Feedback ตัว Workshop ซึ่งส่วนหนึ่งจะเป็นคำถามที่ยังค้างคาใจอยู่ ก่อนหน้านี้ผมคิดว่าจะเอาไว้ตอบบน Group alumni แต่มาคิดอีกทีเอามาตอบบน blog ดูน่าสนุกกว่า แล้วถ้ามีใครมีความเห็นอื่นๆ จะได้ลองแลกเปลี่ยนกันด้วย


Blog แรกเลยเลือกคำถามสนุกๆ โดยคำถามมีบริบทว่า “ถ้าในบริษัทมี UX อยู่คนเดียว คนที่เหลือไม่รู้จัก UX” แล้ว "เราควรเริ่มต้นอย่างไร” เป็นคำถาม

สำหรับการตอบคำถามนี้ผมอยากชวนให้คิดถึงเป้าหมายก่อน แล้วค่อยหาทางเริ่มต้น

สำหรับคนที่ทำงานด้าน UX ผมจะได้เห็นมาหลายแบบ แต่ละแบบก็มีเป้าหมายไม่เหมือนกัน ซึ่งทำให้การเลือกทางเดินเริ่มต้นต่างกันมาก ผมจะลองยกตัวอย่างซัก 3 แบบพอเป็นแนวๆ

แบบแรก เป็นพวกฉายเดียว มีเป้าคือการทำให้ product สร้าง Experience ที่ดีที่สุด พบได้มากในกลุ่ม Startup

คนที่ทำ UX ในกลุ่มนี้จะอยากทำงานสุดๆ มีความคิดความฝันและรู้ว่าควรทดสอบโปรแกรมกับผู้ใช้อย่างไร ดังนั้นกลุ่มนี้จะเริ่มต้นด้วยการทำ User interview เพื่อจะรู้ว่าปัญหาที่ควรแก้ จากนั้นก็ต่อด้วย idea creation, prototype, test แล้ววนไปเรื่อยๆ จนกว่าจะเริ่มลงตัวก็ส่งไปให้ Development ทำงานต่อ แล้วตัวเองก็กลับไปเริ่มต้นใหม่ ถ้า Development ทำเสร็จแล้วก็จะเอาไปทดสอบ ซึ่งทำให้ product ก็จะดีขึ้นอย่างรวดเร็ว (แต่อาจจะมีปัญหาเรื่องกำไร เพราะตามใจลูกค้ามากเกิน)

แบบที่สอง เป็นพวกที่เน้นผลงาน มีเป้าหมายคือการทำให้ product สามารถตอบโจทย์ธุรกิจได้ (สร้าง conversion หรือ เปลี่ยนผู้ใช้เป็นลูกค้า) พบได้มากในกลุ่ม e-commerce

คนกลุ่มนี้แทนที่จะเริ่มต้นที่ลูกค้า เค้าจะเริ่มต้นที่ Stakeholder หรือกลุ่มที่ให้ทุนกับ product เพื่อให้รู้ว่าจริงๆ เค้าทำโปรแกรมนี้เพื่ออะไร เป้าหมายคืออะไร สมติฐานของคนกลุ่มนี้คืออะไร และที่สำคัญคือเค้าคิดว่าประสบการแบบไหนที่สามารถทำกำไรได้ จากนั้นทีม UX ก็มีหน้าที่เอาสมติฐานมาทดสอบกับผู้ใช้ สร้าง persona, user journey แล้วทำ User test (interview, prototype etc.) หลังจากได้ข้อมูลก็กลับไปคุยกับ Stakeholder อีกที ถ้าดูแล้วเพิ่ม conversion หรือ NPS ก็ส่งให้ dev ทำต่อ

แบบที่สาม เป็นพวกเพ้อฝัน มีเป้าหมายคือการทำให้ทุกคนในองค์กรมี User อยู่ในใจ แล้วทุกคนพยายามสร้าง Experience ที่ดีที่สุดให้ลูกค้า พบได้ไม่บ่อยนัก บริษัทที่จะใส่ใจผู้ใช้มากกว่าผลกำไร อย่าง Apple (ก่อนสมัยของคุก)

UX ที่อยู่ในกลุ่มนี้ไม่ได้มีเป้าหมายหลักเพื่อสร้างประสบการณ์ที่ดีที่สุดให้ลูกค้า เพราะเค้าคิดว่า UX ไม่ใช่คนที่รู้จัก Product ดีที่สุด ไม่ใช่คนที่เดาลูกค้าได้ดีที่สุด คนที่ทำได้ดีคือทีมงานที่คลุกคลีอยู่กับ Product มานานต่างหาก UX เป็นแค่คนที่รู้เรื่อง UX เท่านั้น

ดังนั้นสิ่งแรกที่ UX ทำคือการ Interview Team แล้วพา user ให้เข้ามาใกล้ทีมให้มากที่สุด ช่วยทีมงานในการวัดสมติฐานต่างๆ และสร้างความเข้าใจในตัว User ให้กับทีมสอน พาทำกระบวนการต่างๆ ของ UX จนทีมงานสามารถทำได้เอง แล้วทั้งทีมละที่จะเป็นคนสร้างประสบการณ์ที่ดีให้กับผู้ใช้ ไม่ใช่แค่ UX คนเดียว

จะเห็นว่าไม่ว่าจะเป็น UX แนวไหน กว่าที่ Product จะแสดงออกถึงความใส่ใจในผู้ใช้ ก็ต้องใช้ระยะเวลาทั้งนั้น แต่ถ้าไม่เริ่มหนทางก็จะไกลขึ้นเรื่อยๆ
