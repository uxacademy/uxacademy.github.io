บท Lean UX เป็นอีกบทที่คัดออกมาจากหนังสือ Design Mobile App ครับ อยากจะเอามาแชร์ให้ได้ลองวิเคราะห์กันดู

ในองค์กร หรือในบริษัทที่เริ่มต้นพัฒนาโปรแกรมของตนเอง จะพบกับปัญหาในเรื่องการสื่อสารอยู่เสมอ และสิ่งที่เลวร้ายกว่าการพบปัญหาก็คือการไม่รู้ว่าตนเองกำลังพบกับปัญหานั้นเอง จึงทำให้เข้าใจกันไปเองว่าทุกคนสื่อสารเข้าใจสิ่งเดียวกันแล้ว ยกตัวอย่างเช่น ผู้บริหารธุรกิจกำลังคุยกับนักพัฒนาในองค์กร โดยผู้บริหารยกตัวอย่างว่าเขาอยากได้ระบบการขายแบบ Amazon โดยเล่าถึงประสบการณ์ว่าเขาสามารถค้นหาสินค้าได้อย่างรวดเร็วผ่านทางชองค้นหา วินาทีนี้สิ่งที่อยู่ในหัวของนักพัฒนาคือเทคนิคในการค้นหาอย่างรวดเร็ว ฐานข้อมูลที่ต้องใช้ การพัฒนาระบบค้นหาผ่าน cloud หรือเทคนิคการเชื่อมโยง keyword เพื่อให้ผู้ใช้สามารถค้นหาได้อย่างรวดเร็ว แต่สิ่งที่ผู้บริหารคิดกลับไม่ใช่เรื่องนี้เลย เพราะผู้บริหารไม่มีพื้นฐานในทางเทคนิค สิ่งที่เค้าคิดคือช่องค้นหาที่อยู่บนสุด และมีช่องให้เลือก category อยู่ด้านหน้าทำให้ผู้ใช้มองเห็นและค้นหาได้ทันที

เมื่อทั้งสองคนตีความคำว่ารวดเร็วไม่เหมือนกัน บทสนทนาต่อจากนี้จะเริ่มไม่ไปในแนวทางเดียวกัน ทั้งสองคนต่างพยายามแก้ปัญหากันคนละจุด และแม้ว่าจะสามารถหาจุดที่ลงตัวร่วมกันได้ แต่ภาพในหัวของทั้งสองคนก็มีจุดที่ให้ความสำคัญไม่เหมือนกันอยู่ดี

ในที่สุดเราก็ได้ข้อสรุปแล้ว

นี่เป็นจุดเริ่มต้นของความล้มเหลวและงบประมาณที่ต้องทุ่มลงไป ไม่ใช่เพื่อพัฒนาโปรแกรม แต่เพื่อลองผิดลองถูกจนกว่าสองคนนี้จะเข้าใจกัน ในการพัฒนาโปรแกรมโดยเฉพาะกับกลุ่ม Startup เวลาและงบประมาณเป็นสิ่งมีค่ามาก เราไม่ควรสูญเสียงบประมาณไปกับสิ่งนี้ ดังนั้นแทนที่เราจะเริ่มต้นจากมุมมองของแต่ละคนเราควรเริ่มต้นจากมุมมองของผู้ใช้โดยเริ่มจากปัญหาของผู้ใช้ก่อน แล้วค่อยๆ พัฒนาออกมาเป็นแนวทางในการแก้ปัญหา จนสุดท้ายได้เป็นแนวคิดของโปรแกรม กระบวนการแบบนี้จะทำให้มองเห็นภาพได้ง่ายกว่า รวดเร็วกว่าและประหยัดกว่าเพราะทั้งสองคนมองภาพจากมุมมองเดียวกันอยู่ตลอดเวลาทำให้ไม่ต้องเสียเวลาในการสื่อสารกันมากนัก

ในที่สุดเราก็ได้ข้อสรุปแล้ว

ลองนึกถึงการแก้ปัญหาโดยไม่ใช้วิธีการนี้ แต่เราให้คนทำ UX ไปหาข้อสรุปมาให้เรา คนที่ทำ UX ก็ต้องเริ่มต้นกระบวนการหาความต้องการของผู้ใช้โดยอาจจะใช้กระบวนการ User Centered Design หรืออื่นๆ จนกระทั้งได้วิธีการแก้ปัญหาและรูปแบบของโปรแกรม ซึ่งทั้งหมดนี้ยังถือว่าใช้เวลาไม่มากนักถ้าเทียบกับเวลาที่ UX ต้องบันทึกสิ่งที่ศึกษามาเป็นเอกสาร แล้วนำมาถ่ายทอดให้กับผู้บริหารและนักพัฒนาโปรแกรม แถมก็ยังไม่แน่ว่าเมื่อทำแล้วจะได้คำตอบที่ถูกต้องเพราะเรายังไม่รู้ว่าผู้บริหารเห็นด้วยกับวิธีนี้หรือไม่ และไม่รู้ว่านักพัฒนาของบริษัทสามารถทำได้จริงหรือเปล่า สุดท้ายกว่าจะได้เริ่มทำงานแนวคิดของเราก็อาจจะโดนคนอื่นแย่งทำไปแล้ว

Lean UX เข้ามาแก้ปัญหานี้ โดยนำเสนอแนวคิดที่ออกแบบมาเพื่อให้เราสามารถแก้ปัญหาได้เร็วขึ้น ใช้เวลาสื่อสารกันน้อยลงแต่ได้ประสิทธิภาพมากขึ้น แถมลดความเสี่ยงในการลงทุน และที่สำคัญคือทำให้เรารีดความสามารถของเราออกมาได้เต็มที โดยแนวคิดของ Lean UX ได้รับอิทธิพลมาจาก Agile Development และ Lean Startup โดยในหนังสือ Lean Ux ได้บอกว่า

“Lean UX Inspired by Lean Startup and Agile development theories, it's the practice of bringing the true nature of design work to light faster, in a collaborative, cross-functional way with less emphasis on deliverables and greater focus on the actual experience being designed.”

ที่เน้นคำว่า true nature และ actual experience เพราะธรรมชาติของการออกแบบนั้นคือการค้นหาประสบการณ์จริงๆ ของผู้ใช้ ไม่ใช่ประสบการณ์ที่เราคิดขึ้นมาเอง ยิ่งเราพัฒนาสินค้นของเราได้เร็วแค่ไหนเราก็ยิ่งสามารถนำไปทดสอบได้เร็วเท่านั้น และยิ่งทดสอบได้เร็วมากขึ้นก็ช่วยให้เราได้รู้ประสบการณ์ของผู้ใช้ต่อโปรแกรมของเราได้เร็วขึ้นด้วย

ลองย้อยกลับมาดูแผนภาพของแนวคิด Lean Startup กัน เราจะเห็นว่า กระบวนการนี้ให้เรานำสินค้าของเราไปทดสอบกับผู้ใช้ แล้วนำผลลัพท์ที่ได้กลับมาเรียนรู้และพัฒนาขึ้นมาใหม่เพื่อนำไปทดสอบกับผู้ใช้อีกครั้ง สิ่งที่สำคัญที่ Lean Startup บอกก็คือการเร่งรอบทำให้กระบวนการนี้หมุนให้เร็วที่สุด ไม่จำเป็นว่าหนึ่งรอบจะต้องใช้เวลาหลายอาทิตย์ บางครั้งหนึ่งรอบอาจจะใช้เวลาแค่วันเดียว หรือในหนึ่งวันอาจจะทำได้ถึงสองรอบก็ได้

ต้นแบบ Palm pilot

คนที่ทำ Lean UX จะหาทางพัฒนาสินค้าของตัวเองออกมาให้เร็วที่สุด เช่นเขียนลงบนกระดาษ แล้วนำไปทดสอบกับผู้ใช้ก่อน จากนั้นกลับมาเขียนใหม่แล้วเอาไปทดสอบอีกครั้ง แค่ทดสอบไปสองครั้งเราจะได้ข้อมูลกลับมามากมาย แทนที่จะต้องเสียเวลาไปพัฒนาโปรแกรมที่สุดท้ายอาจจะไม่สามารถใช้งานได้ ก็สู้พัฒนาบนกระดาษก่อนแล้วนำไปทดสอบ ยอมเสียเวลาตรงจุดนี้จนมั่นใจจึงค่อยพัฒนาโปรแกรมก็ได้

เวลาที่ทำ Lean UX เราจะเร่งรอบให้ได้มากที่สุด นั่นแปลว่าคนที่ทำ Lean UX จะต้องฟิตมากๆ เพราะเราคิดอยู่ตลอดเวลาว่าทำอย่างเราให้สินค้าของเราเสร็จพร้อมที่จะนำไปทดสอบอยู่เสมอยิ่งถ้าเราย้อนมาดู Agile Manifesto เราจะเห็นชัดเจนมากขึ้น
