@วันนี้ก็เก็บสะสมความรู้จัก facebook/groups/ux.in.th อีกนะครับ คำถามวันนี้ได้จากคุณ @darissa.ble

ขอความเห็นหน่อยค่ะ มีการวางปุ่ม 3 แบบตามรูป เพื่อนๆคิดว่าการวางแบบไหนทำ ให้ผู้ใช้งานรู้สึกว่าใช้งานง่าย และทำให้เกิดการกดผิดน้อยที่สุด ( open กับทุกความคิด และเหตุผลค่ะ )

ได้ความคิดเห็นมาสี่สิบกว่าข้อความ ผมขอคัดบางอันออกมานะครับ

@Thammanoon Soparat จำไว้เสมอนะครับว่าปุ่ม no จะไม่มาก่อน yes ปุ่ม cancel ก็เช่นกัน และ next กับ back ก็เหมือนกัน อ้างอิงตามหลักความเป็นจริงนะครับ

@Vadi Kumrungsee @Thammanoon Soparat พูดถูกครับ ตามมุมมองผมนะครับ อันนี้ผมขอแสดงความเห็นหน่อย การที่ปุ่ม Y หรือ N จะมาก่อน มันขึ้นอยู่กับว่าเราอยากให้เขาทำอะไรมากกว่าครับ เช่นถ้าเราไม่อยากให้เขาลบอันนี้จริงๆ เราก็ควรจะ Convince ให้เค้ากด No ครับ


แบบนี้น่าจะหมายถึงปุ่มไหนสำคัญหรือเรียกว่าเป็น Primary Button ก็ให้เอาไว้ทางซ้าย

ต้องดูด้วยว่าเค้าอ่านจากซ้ายไปขวา หรือขวาไปซ้าย แต่เนื่องจาก interface ปัจจุบันเค้าเรียงซ้ายไปขวาหมด ดังนั้นแม้แต่ประเทศที่อ่านหนังสือขวาไปซ้าย ก็น่าจะเคยชินกับการเรียงปุ่มแบบเอา Primary ไว้ซ้ายกัน

ในที่นี้ต้องยกเว้นว่าเราต้องการส่งต่อ message ไปยังหน้าถัดไป เช่นปุ่ม "Next >" หรือ "Continue >" เราจะไม่ได้คำนึงถึงว่าคนจะมองซ้ายหรือขวาก่อน แต่น่าจะเอาตาม Transition หรือการเปิดหน้าหนังสือมากกว่า เพราะมันไม่ใช่ปุ่มที่ตัดสินใจเลือก Yes/No แต่เป็น Action ที่บอกว่าถัดไป

อีกหนึ่งความเห็นจากคุณ @Sansern Wuthirat ว่าให้ใส่ action ลงไปในปุ่มที่ตัดสินใจด้วย

2 ครับ Dialog แรกน่าจะเปลี่ยนคำว่า Yes/No เป็น Cancel / Delete หรือ Yes, delete file ผู้ใช้เข้าใจง่ายดีครับ

คุณ @Karun Warapongsittikul มี reserch รองรับด้วย

 แบบที่สองครับ ผมเคยลองทำ blind test เรื่องนี้กับกลุ่มตัวอย่าง 200คน โดยให้เค้าลองเลือกปุ่มที่จะกดเอง โดยไม่มี label บอกบนปุ่ม 80% กว่าๆเลือกกดแบบที่สองเพราะมัน match กับnatural mapping ของการกด next ครับ

อีกหนึ่งความเห็นต่างจากคุณ @Phuluang Hanpipat

ผมกลับมองว่า ต่อให้เป็น Dialog ของการ Delete ก็ควรวางปุ่ม Yes หรือ Comfirm ไว้ทางขวาอยู่ดีนะครับ เพราะจุดประสงค์ของการกดปุ่ม Delete ก็คือต้องการ Delete ดังนั้น step หลังจาก comfirm คือการ Delete จริงๆ ไม่ใช่เหรอครับ?

โดยเฉพาะอย่างยิ่ง ถ้าหากเลือกแนวทาง 2 (มีการวาง next, yes ไว้ทางขวา) ตลอดทั้งไซต์แล้ว หากจะสลับที่เพราะเคส Delete คงงงกันน่าดู เว้นแต่ว่าจะ Hightlight ไว้ที่ back หรือ no ยังพอถูไถครับ

สุดท้ายก็ต้องทดลองกันต่อไปนะครับ จริงๆ ผมมองว่าการตอบคำถามนี้มันมีบริบทของการใช้งานประกอบด้วยมากมายทั้ง persona และ workflow แต่ที่สำคัญในหนึ่ง web/app น่าจะชัดเจนว่ามีหลักในการตัดสินใจอย่างไร ไม่ใช่ปุ่ม Yes เหมือนกัน แต่อยู่ซ้ายทีขวาทีครับ
